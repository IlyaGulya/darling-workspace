# revision as an ERROR, unless its name or path is listed here. This is what would have caught #90
# (xnu silently on an experimental perf branch instead of the manifest revision).
#
# One project NAME (as in `west list`) or PATH per line; '#' starts a comment. Declare only
# INTENTIONAL drift for current dev work, and note WHY. Remove a line when the project is reset to
# its manifest revision.

# one." For a KNOWN-GOOD BOOTABLE closure build, the closure projects must be at their manifest
# revisions (e.g. xnu: the perf/shmem-ring-guest branch does NOT boot — #90). Reset to manifest-rev
# for deployable builds, then restore dev drift.
