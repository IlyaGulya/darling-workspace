# Red-Test Coverage Audit - 2026-07-08

Scope: committed patch profiles `arch`, `homebrew`, and `perf`.

Rule used for this audit:

- `covered`: the patch itself commits a runnable regression/contract/gate test, or the same Bead has a committed follow-up test patch in the same patch profile.
- `runtime-only`: the patch message or Bead records runtime/manual validation, but no portable committed red-test lives with the patchset.
- `gap`: no committed red-test was found in the patch profile.
- Pure documentation/comment-only patches are listed separately when applicable.

This audit is intentionally strict. A historical local experiment, manual gate result, or commit-message claim does not count as "committed with the fix" unless the test artifact is in the patchset.

## Follow-up Recommendation

Prioritize adding tests in this order:

1. High-risk runtime correctness gaps in `homebrew`: signal/wait/fork/checkin patches (`dar-gwn.*`, `dar-q95.8`, `dar-q95.13`, `dar-q95.15`, `dar-gwn.6`).
2. Simple syscall/API contract gaps in `homebrew`: `select`, `priority`, `socket ioctl`, `getattrlist`, `getattrlistbulk`, `fd-guard`, `sdk-homebrew-detection`.
3. Promote `experiments/e-union` into a committed patchset harness or explicitly mark E-UNION as experimental runtime-only.
4. Perf/dyld/libunwind gaps need build/runtime guard tests rather than static grep-only tests.
